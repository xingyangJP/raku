# 営業訪問前AIコーチ 仕様（MVP）

## 背景と目的
- 顧客訪問前に新人営業が「何を聞くべきか分からない」状態を防ぎ、要件抜けを減らす。
- AIがゴール確認→優先質問提示→質問リライト→未決チェック→1ページ要約を支援。
- MFクラウド請求書連携案件を主対象に、PGが設計・見積に入れる粒度で情報を揃える。

## 想定ユーザ/利用シーン
- 利用者: 営業（新人含む）、プリセールス、PM。
- シーン: 顧客訪問前の準備、訪問後の振り返り/補足質問作成、顧客共有資料ドラフト生成。

## 入力/出力（MVP）
- 入力1: 今日決めたいこと/聞きたいこと（自由入力）。
- 入力2: 過去議事録URL or 要約（任意、将来は顧客別RAG検索）。
- 出力A: 優先度付き質問（件数はAIが提案、チェックボックス＋追記欄）。
- 出力B: 「今日決め切る3点」と「決まらない場合の持ち帰り先」。
- 出力C: 質問リライト（営業の質問→深掘り版に書き換え）。
- 出力D: 面談後の抜けチェックと追加質問案。
- 出力E: 1ページ要約ドラフト（ビジネス要求/やる・やらない/未決事項/次アクション）。

## フロー（画面遷移イメージ）
1. **コンテキスト取得**（任意）: 過去議事録要約を貼る → 目的/未決を抽出（RAGは後付け）。
2. **ゴール確認**: 「今日は何を決めたい？」入力 → カテゴリ分類（機能/非機能/制約/予算/スケジュール）。
3. **優先質問提示**: ゴールに合わせて質問を生成（件数はAIが提案）。未回答だと致命的な項目は赤表示。
4. **質問リライト**: 営業が質問案を入力 → AIが現状→理想→制約→例外→優先順位の順に深掘り版を返す。
5. **合意事項の明示**: 「今日決め切りたい3点」を提示。決まらなければ持ち帰り先（顧客/社内）を明記。
6. **面談後チェック**: メモを貼る → 抜けやすい項目（権限、移行、同期頻度、SLA、例外）をハイライトし、追撃質問を生成。
7. **1ページ要約生成**: ビジネス要求 / やる・やらない / 未決事項 / 次アクションをMarkdownで出力。

## 優先質問テンプレ（MF連携案件例）
1) OAuth/接続: MF請求書のクライアントID/シークレットとリダイレクトURI登録は可能か、いつ準備できるか。  
2) 同期対象・方向・頻度: 品目/取引先/部門を片方向/双方向のどれで、手動/自動/差分のどれで同期するか。  
3) 承認フロー: 誰がどの金額/条件で承認・差戻しするか。例外、期限、申請不可条件は。  
4) 見積→請求変換: 変換タイミングと例外、MF側修正時の扱い。  
5) PDF/帳票: レイアウトや項目のカスタム要否。既存テンプレが流用できるか。  
6) データ移行: 既存品目/取引先/過去見積・請求を移行するか。クリーニング要否。  
7) 非機能/運用: 想定ユーザ数/同時利用、稼働時間、障害連絡SLO、操作ログ保持期間。

## UIラフ（コンポーネント単位）
- テキスト入力: 「今日決めたいこと」。
- テキストエリア: 過去議事録URL/要約（任意）。
- 質問リスト: チェックボックス＋追記欄でAI提案の件数を表示。未回答必須は赤ラベル。
- リライト欄: 営業の質問入力 → リライト結果表示（並列表示）。
- 合意トラッカー: 「今日決める3点」「持ち帰り先」カード。
- 面談後貼り付け欄: メモ入力 → 抜け指摘＋追加質問表示。
- 出力表示: 1ページ要約をMarkdownプレビュー（コピー/保存ボタン）。

## データ・RAGの考え方（後付け可）
- ストア単位: 顧客ごとにベクトルストアを分ける。ドキュメント種別タグ（議事録、メール要点、決定事項、未決事項）。
- 検索キー: 目的カテゴリ（機能/非機能/制約/予算/スケジュール）＋機能領域（MF連携、承認、同期、帳票）。
- プライバシー: チーム権限でフィルタ（営業チームのみ等）。

## 非機能・運用（MVPの最低ライン）
- レスポンス: 3秒以内目標（RAGなし前提）。RAGありは10件以内のコンテキストに圧縮。
- ログ: 入力/出力を案件ID単位で残し、個人情報はマスク（氏名/メール/電話）。
- エラー時: 質問テンプレのデフォルトセットを返してフォールバック。

## スコープ外（MVP）
- 複雑なカスタム権限管理、マルチテナント管理画面。
- MF以外の会計/ERP連携。
- PDFレイアウトエディタ。
- 自動スケジューリング/招待メール送信。
