# RAKUSHIRU Cloud LP設計書（管理者向け）

> ステータス: 提案資料（LP設計ドラフト）。運用仕様の正本ではありません。

## 目的
- 管理者（部門責任者・経営層）が「何ができ、どんな効果があるか」を素早く理解し、導入相談/デモ申込を行う。
- Money Forward（マネーフォワード）クラウド請求書連携を軸に、「承認と会計をつなぐ」価値を強調する。

## ターゲット
- 見積・請求・日報・原価管理が分断され、数字がリアルタイムで揃わない組織
- MFクラウド請求書を利用中、あるいは検討中で、社内承認やマスタ管理を統合したい管理者
- 内部統制/監査対応で証跡を求められる企業

## 管理者目線の主要ベネフィット
- 数字の一元化：見積→請求→原価をMFと双方向同期し、ダッシュボードでリアルタイム可視化
- 承認漏れ防止：承認フローと日報リマインドで入力/承認の抜けを削減
- 監査対応：操作ログ・承認履歴・PDF/リンクで証跡を提示しやすい
- 導入容易：既存のMF利用を活かし、CSV/Excel連携も併用可能
- AI活用で初動を高速化：要件整理チャットとAIドラフト見積で、手戻りと作業時間を削減

## ページ構成案
1. ファーストビュー  
   - キャッチ：「見積・請求・原価を“承認済みの数字”で、MFとリアルタイム連携」  
   - サブコピー：「承認フロー、品目/取引先マスタ、MF同期までワンストップ。管理者の集計負荷を削減。」  
   - CTA：デモを予約 / 相談する
2. 課題提示（3〜4項）  
   - 「日報/見積がバラバラで粗利が読めない」「承認待ちが滞留」「MFとの品目・取引先同期が手作業」「証跡要求に時間」
3. ソリューション/主要機能（管理者視点）  
   - ダッシュボード：承認待ちタスク、売上/粗利サマリ、MF同期状況を一画面で  
   - 見積・請求：承認フロー付き見積、MFへの発行/変換、PDF取得  
   - マスタ同期：取引先・品目をMFと自動/手動同期、差分アップサート  
   - 日報/原価：リマインド・未提出警告、レート管理、原価/粗利の分析表示  
   - 監査・証跡：操作ログ、承認履歴、レポート出力（PDF/リンク）  
   - 連携：CSV/Excel入出力、API連携（拡張余地）
   - AI支援：要件整理チャット＋AI見積ドラフト生成で、初動の要件定義と工数見積を支援
4. 効果訴求（例示数値）  
   - 未提出/未承認の削減、締め処理時間短縮、コスト超過の早期検知、要件整理〜ドラフト作成時間の短縮
5. ワークフロー図示  
   - 「入力→承認→MF同期→ダッシュボード可視化→エクスポート」の流れ
6. セキュリティ/統制  
   - ロール権限、操作ログ、SSL、バックアップ、監査向けエクスポート
7. 料金・導入の流れ（シンプル）  
   - 相談→トライアル→本番導入
8. FAQ（管理者が気にする点）  
   - 権限分けは？既存データ移行は？MF連携の要件は？監査証跡は？
9. フッターCTA  
   - デモ予約 / 資料請求

## ハイライトすべき具体機能（リポジトリ準拠）
- MFクラウド請求書との連携  
  - 見積一覧・編集でMF見積の取得/発行、PDF表示、請求への変換  
  - 請求一覧でMF請求とローカル請求を統合表示し、リンク/編集/ダウンロード  
  - 取引先・部門同期をダッシュボードで自動実行、手動再同期ボタンあり  
  - 品目マスタをMFに差分同期（作成/更新/削除）
- AIによる要件整理・見積ドラフト  
  - 要件整理チャット：不足情報を質問リスト化し、機能/非機能をMarkdown章立てで整理（`README_AI_ESTIMATE.md`, `docs/AI_ESTIMATE.md`）  
  - AI見積ドラフト生成：整理済み要件からシステム開発向け明細を自動生成（PM必須の有無も反映）  
  - 生成結果はモーダルでプレビューし、「すべて採用/一部挿入/再生成」を選択可能  
- 承認付き見積ワークフロー  
  - 承認ステップ管理（並び替え・追加/削除）、承認/申請取消  
  - 粗利率や条件不足時の申請不可制御（社内ルール反映）
- 原価/粗利・事業区分集計  
  - 受注確定フラグで予実管理、売上ランキングや事業区分別集計を表示
- UI/UX  
  - Inertia + React + shadcn/ui + Tailwind のモダンUI  
  - ダッシュボードに「やることリスト」と同期ステータスを表示

## FAQサンプル
- 権限は分けられる？ → ロール/権限で閲覧・編集を制御可能  
- 既存データは移行できる？ → CSV/Excel インポート対応、MF側マスタは同期可  
- MF連携の要件は？ → Money Forwardクラウド請求書のOAuth設定（client ID/secretとリダイレクトURI登録）が必要  
- 監査証跡は？ → 操作ログ/承認履歴/PDF・外部リンクの保持で提示が容易
