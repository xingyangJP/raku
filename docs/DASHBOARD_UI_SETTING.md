# ダッシュボードUI設定書（経営者向け・セクション分割版）

## 1. 目的
- 経営者が「売上・粗利・仕入・工数・資金繰り」を同一画面で判断できること。
- 事業構造に合わせて、以下4セクションで視認性を上げること。
  - 総合
  - 開発セクション（受託開発）
  - 仕入れ販売セクション
  - 保守セクション

## 2. 画面構成
- 画面上部: グローバルフィルタ
  - 期間（月次）
  - 表示基準（納期ベース固定）
  - 予算/実績の定義表示（予算=見積、実績=受注確定）
- 画面中部: セクション切替タブ
  - `総合` `開発` `仕入れ販売` `保守`
- 画面下部: 選択セクションのカード群と明細テーブル

## 3. セクション定義

## 3.1 総合
- 目的: 全社の予実と資金繰りを俯瞰する。
- 表示KPI:
  - 売上（予算/実績/差異）
  - 粗利（予算/実績/差異）
  - 仕入（予算/実績/差異）
  - ネットキャッシュフロー（支払予定・回収予定・回収実績）
- 表示テーブル:
  - 月次予実一覧（売上/粗利/仕入）
  - 月次キャッシュフロー（支払予定/回収予定/回収実績/ネット）

## 3.2 開発セクション
- 目的: 受託開発の収益性とキャパ運用を管理する。
- 対象データ:
  - 商品 `business_division = fifth_business`
- 表示KPI:
  - 売上（予算/実績/差異）
  - 粗利（予算/実績/差異）
  - 計画工数（人日）
  - 計画稼働率
  - 空き工数（計画）
  - 計画生産性（粗利/人日）
- 表示テーブル:
  - 月次 工数計画（人日）
  - 仕入内訳（物品/工数原価）

## 3.3 仕入れ販売セクション
- 目的: 先出し資金と回収タイミングを重視して収益を管理する。
- 対象データ:
  - 商品 `business_division = first_business`
- 表示KPI:
  - 売上（予算/実績/差異）
  - 粗利（予算/実績/差異）
  - 仕入（予算/実績/差異）
  - 在庫・回収関連の注意指標（将来拡張）
- 表示テーブル:
  - 月次 仕入・売上予実
  - 月次 キャッシュフロー（支払先行を強調）

## 3.4 保守セクション
- 目的: 保守契約の継続売上と工数消化の健全性を管理する。
- 対象データ:
  - 保守関連明細（定義は商品分類または明細属性で判定）
- 表示KPI:
  - 保守売上（予算/実績/差異）
  - 保守粗利（予算/実績/差異）
  - 計画工数（人日）
  - 計画稼働率
  - 空き工数（計画）
  - 計画生産性（粗利/人日）
- 表示テーブル:
  - 月次 保守予実（売上/粗利/計画工数）
  - 契約別または顧客別 保守実績（将来拡張）

## 4. 工数管理（開発・保守共通）
- 工数の単位は人日に統一する。
- 換算ルール:
  - 人日: そのまま
  - 人月: 20人日換算
  - 人時/時間: 8時間=1人日換算
- 予算工数:
  - 見積明細から算出（対象セクションのみ）
- 実績工数:
  - 本仕様では未使用（日報未連携前提）
- 不足/過剰の表現:
  - 空き工数 = キャパ - 計画工数
  - 過負荷（マイナス）時は警告色で表示

## 5. 情報設計（カード順序）
- 1段目: セクションKPI（売上・粗利・仕入/工数）
- 2段目: 工数詳細（開発・保守のみ）
- 3段目: 月次テーブル（予実）
- 4段目: キャッシュフローとランキング/注意指標

## 6. UIルール
- 金額は `¥` 表記、3桁区切り。
- 工数は小数1桁の人日表示。
- 差異は `実績 - 予算`。
- 色の意味:
  - 良化: 緑
  - 悪化: 赤
  - 中立: グレー
- セクションを切替えても期間フィルタは保持する。
